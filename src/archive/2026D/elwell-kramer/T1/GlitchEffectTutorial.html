The glitch effect is based on a form of datamoshing, a technique that recycles old pixels. It’s based on glitches that can occur with video compression but can be caused intentionally as well. In a real-time game engine, it works slightly differently but is still possible.

Unreal Datamoshing Overview: https://youtu.be/Uapad3pVxBY?si=vkA8cTOHOfVcqIDD 

Outside of preexisting plugins, there are few resources for datamoshing in Unreal. That overview video was the closest to a tutorial, and even it said to use this plugin and look at the pieces of it. But I didn’t want to pay for it, so I tried to figure it out myself.

I eventually found a reddit post which had an image of a datamosh shader graph... the only problem was it was for Unity, while I was working in Unreal.

I tried to copy it over and translate, but even after finding equivalent nodes, it still required alteration due to differences in how the two engines work. After some alterations, experimentation, and improvements, I ended up with this:

It ended up not quite being the datamoshing I was looking for, but it created a very interesting effect that can work well for a glitch or cloaking effect! Now, let’s go into some of the details on how it works.

 

Material & Post Processing Setup

Create a new material, give it a name, and open the material graph. In the details panel, set Material Domain to Post Process and Blendable Location to Scene Color Before Bloom. We’ll be building our shader in this material graph.

We also want a material instance so we can control the parameters we create. Right click on the material in the content browser and select Create Material Instance. When naming your material and instance, you want it to be clear which is which - convention is to give your material the prefix MM for Master Material, and the instance MI for Material Instance.

Now we need to make sure our scene can use the post-process shader. Create a Post Process Volume in the scene, and go to its details panel. Turn on Infinite Extent Unbound.

Next, go to Rendering Features -> Post Process Materials -> Array and click the plus to add an element. Drag your material *instance* into it.

Later, once we have parameters, you’ll be able to control and tweak them from the material instance. But for now, we want to go back to the master material to work.

Motion Vectors

First up are motion vectors. Unreal stores data on how fast the parts of the scene are moving relative to the camera, which we can access via SceneTexture:Velocity. We then split out its R and G components and append them to make a Vector 2. We multiply this by a scale parameter to control the effect’s strength, and add it to the scene’s UVs, accessed via TexCoord[0]. This gives us the basis of the effect, which will distort the screen based on the camera/objects’ movement.

“Color” (Scene Textures)

This is where we put things together to get the effect to show up. We need two SceneTexture nodes, both set to PostProcessInput0, to get our post process scene. Into one of them we simply plug TexCoord[0] into the UV input, which will get us the regular scene without any post processing effects. To the other, we plug the result of out motion vector section - the distorted UVs we created earlier. Lerp these two scene textures together, and we have our distortion effect! With the alpha input of the lerp, we can control how intense the effect is. If we plug the result of this lerp into the emissive color channel of the material, we get this:

Which is very disorienting, and not *quite* what we want. Let’s add some more.

Pixelated Noise

Since our effect is controlled by a lerp, we can mask it out using the lerp’s alpha channel to make it appear in blocky pixels. First we need a noise scale parameter to control the pixels’ size, or they’ll be way too small. 0.01 works well, but we need a Vector3 for what we’re doing, so plug the parameter into three channels of an AppendMany node to do so (or if you don’t want square pixels, you can make the parameter a Vector3 instead). Then we need the Absolute World Position (just search ‘World Position’ to find it) to tell us where our pixels should go. Multiply it by our scale, then round the result, and then divide that by the scale again to get things a bit distorted.

Plug this result into the World Position input of a Noise node, then multiply that by a parameter to control the noise’s intensity. Lastly, I set up a switch statement controlled by a parameter to toggle between using the noise as a mask and just having a mask value of 1. The result of this plugs right into the alpha channel of our lerp from earlier!

Much glitchier, and easier on the eyes too. But we hardly want the entire scene to look like this, so we’ll need to mask out certain objects.

Stencil Mask

First, make sure custom stencils are enabled in project settings!

Here we’re basically telling the post processing to only apply on objects that we designate. Get a SceneTexture:CustomStencil node and mask out the G channel, then divide it by 255. Create a BitMask and plug the result of the divide into the BitMask channel, then a parameter into the Bit channel. This number is the “layer” of objects to which we want to apply the effect. In our scene, objects default to a CustomDepth Stencil Value of 0 - set the parameter in our material to the channel you want to use (I used 1), then in the details panel set the CustomDepth Stencil Value of objects we want the effect on to this same number.

While we’re here, we also need to check on Render CustomDepth Pass (and if you don’t want to come back to this panel later, uncheck the two checkmarks above it - but we’ll get to that later).

If we want some objects to block the glitch effect, we can make a second bitmask with the number of another channel (i.e. 2 in my example) and subtract that from the first bitmask. However, we’ll also later be adding an option for all objects to block it if they’re in the way.

Regardless, we then Clamp this value between 0 and 1 so it can hook up to an alpha channel, and make another switch statement to toggle the use of the stencil mask (it’s set to off by default in my screenshot, but you can set it to true if you want the effect to use it by default).

To use the mask, we need another lerp. We want the alpha channel to be our stencil mask result, the B channel to be our lerp from earlier, and our A channel to be our regular scene texture. Now, we can mask out objects to use the glitch effect!

Here’s a cube using the stencil mask. I also turned up my VelocityScale parameter from 2 to 20 for this so that the effect’s more obvious in the following gifs. Additionally, we probably don’t want the object’s normal material still showing, so let’s remove it from the main render pass. In the object’s details where we enabled the CustomDepth Pass earlier, uncheck “Render in Main Pass” and “Render in Depth Pass”.

Now we’re getting somewhere!

Visibility Through Objects

There’s still one more problem with our stencil mask: it restricts the effect to certain objects, but it doesn’t stop it if there’s an object between the camera and the affected object.

We could set all non-glitch objects to the channel specified in our second bitmask from the stencil mask section, but there’s a more elegant solution.

Subtract SceneTexture:SceneDepth from SceneTexture:CustomDepth, then clamp between 0 and 1 and invert with a 1-x node. This removes areas of the stencilled object blocked by other objects in the scene. Add another parameter controlled switch statement if you want to be able to toggle visibility, then plug it into the alpha channel of yet another lerp (with A as the base scene texture and B as our previous lerp, as before). Plug that into our material’s Emissive Color channel, and we should be good to go!

Bonus: Blueprint for Ease of Use

Right now, there’s a lot of settings we need to change for each object we want to have the effect. Luckily, we can make it easier on ourselves with a blueprint.

Create a new blueprint, open it up, and in the viewport view, add a static mesh to the components.

Go into the static mesh’s details and set things to how we need them to be as explained earlier:

Compile and save, then go back to the scene. We can drag this blueprint into the scene however many times we want and control it via the standard transforms. If we want to change what the mesh actually is, click on StaticMesh in the details panel and simply drag in the new mesh.

Notably, don’t use this same blueprint for characters or objects that are blueprints for other reasons - set the correct settings within those blueprints!
